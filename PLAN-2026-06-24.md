**Code consequence**: we can't hard-code one accel axis as vertical.

(smoothed). This subsumes part of OSI-023.

### B. OSI-023 v1 — gravity-projected vertical in `waytrace_analysis.py`

Add `compute_gravity_vertical(accel, fs)` that:

1. Low-passes accel at ~0.25 Hz to estimate the running gravity unit
   vector `g_hat(t)` (this tracks the slow phone reorientation —
   resting flat, swinging during push, flat again).
2. Projects raw accel onto `g_hat(t)` for "vertical world component".
3. Subtracts `|g|` → vertical-acceleration signal a_v(t).
4. Returns a_v(t) ready for the existing Wk band-pass / VDV / ISO 8608
   chain.

This makes the analysis robust to any mounting orientation **without
needing to know which axis is vertical in advance**. Bonus: it works
for old chair files too — gravity then sits on Y, projection picks Y.
